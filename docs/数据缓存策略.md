# 数据缓存策略

## 1. 核心业务缓存 (需新增)

### 1.1 待上传任务队列 (Upload Queue)
**Key**: `upload_queue`
**作用**: 保证弱网环境下照片不丢失，支持断点续传和自动重试。这是“随拍随存”体验的核心。
**数据结构**:
```json
[
  {
    "id": "uuid-v4",
    "localPath": "wxfile://tmp/...",  // 临时文件路径
    "albumId": "album-123",           // 目标相册
    "status": "pending",              // pending | uploading | failed | success
    "retryCount": 0,
    "createdAt": 1715673600000
  }
]
```

### 1.2 相册列表快照 (Albums Snapshot)
**Key**: `albums_snapshot`
**作用**: 首页秒开体验。先展示缓存数据，后台静默刷新最新数据。
**数据结构**:
```json
[
  {
    "id": 1,
    "name": "旅行",
    "coverUrl": "https://...",
    "photoCount": 42
  }
]
```

## 2. 用户体验缓存 (已实现/部分实现)

### 2.1 用户身份与信息
**Key**: `access_token`
*   **状态**: ✅ 已实现
*   **作用**: API 请求鉴权。

**Key**: `userInfo`
*   **状态**: ✅ 已实现 (settings.js)
*   **作用**: 避免每次冷启动都调用 `/me` 接口，提升侧边栏/个人页显示速度。

### 2.2 偏好设置
**Key**: `settings_preferences` (目前分散在 `autoCompress` 等 key 中，建议合并)
*   **状态**: ⚠️ 部分实现
*   **建议结构**:
```json
{
  "autoCompress": true,    // 是否压缩上传
  "saveToLocal": false,    // 拍完是否同时也存一份到手机相册
  "defaultAlbumId": null   // 默认上传相册
}
```

## 3. 临时与系统缓存

### 3.1 分享访问凭证
**Key**: `current_share_token`
*   **状态**: ✅ 已实现 (share.js)
*   **作用**: 记录用户通过分享链接进入时的 Token，用于后续 API 权限校验。

### 3.2 微信系统缓存
*   **图片缓存**: 微信 `image` 组件会自动缓存网络图片，无需业务层干预。
*   **文件清理**: 需定期检查 `wx.getSavedFileList`，清理无用的临时上传文件，避免超过 10MB/200MB 限制。
